# 🗺️ DriftMap Integrated Design Manifest (WEAPONIZED)

This manifest harmonizes elite UI/UX skills with the DriftMap Engineering workflow. Every frontend action MUST adhere to these standards.

## 🛠️ Integrated Design Stack
1.  **UI/UX Intelligence (`ui-ux-pro-max`):** 161+ color palettes, Glassmorphism, Bento Grids, and Accessibility (WCAG 2.1).
2.  **Standards Enforcement (`web-design-guidelines`):** Continuous auditing against Vercel's Web Interface Guidelines.
3.  **Prototyping Engine (`stitch-design`):** High-fidelity AI screen generation via Stitch MCP.
4.  **Performance Optimization (`vercel-react-best-practices`):** Zero-waterfall data fetching and bundle optimization.
5.  **Iterative Loop (`stitch-loop`):** Autonomous "baton-passing" for continuous UI refinement.

## ⚖️ Foundational Mandates
- **Accessibility First:** Minimum 4.5:1 contrast, 44x44px touch targets, and full screen-reader support.
- **Visual Aesthetic:** Prefer **Glassmorphism** for data panels and **Bento Grids** for dashboard layouts.
- **Performance:** No client-side waterfalls; use `Suspense` and server-side fetching patterns.
- **Consistency:** All styles must be derived from `.stitch/DESIGN.md` if present.

## 🚀 Execution Workflow
1.  **Design System:** Always start with `python3 skills/ui-ux-pro-max/scripts/search.py --design-system`.
2.  **Prompting:** Use the Stitch Prompt Enhancement Pipeline for all high-fidelity requests.
3.  **Audit:** Run `web-design-guidelines` before every deployment to Cloudflare or Vercel.
4.  **Loop:** Maintain `.stitch/next-prompt.md` to ensure UI iterations never stall.
