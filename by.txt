For a regression/shadow testing tool like this, you want a codebase that:

- Has **pure-ish functions** with clear inputs and outputs
- Has **many released versions** to diff against
- Is large enough to surface real behavioral divergence

**Top pick: Prettier**
`github.com/prettier/prettier`

Why it's ideal:
- Input is source code, output is formatted code — completely deterministic
- 400+ releases means tons of version pairs to compare
- Real-world behavioral diffs exist between minor versions (formatting rules change)
- You can feed it 10,000 JS/TS files from any open repo and get instant, diffable output
- No side effects, no network calls — pure transform

**Runner-up: Babel**
`github.com/babel/babel`

- Transpiles JS → JS, output is diffable text
- Plugin system means config changes = behavioral changes
- Good for testing how your tool handles config-driven divergence

**Runner-up: ESLint**
`github.com/eslint/eslint`

- Returns structured JSON lint results per file
- Config changes between versions produce measurable output diffs
- Easy to automate at scale

**For a heavier stress test: CPython**
`github.com/python/cpython`

- Run stdlib functions across two Python builds
- Output diffs on things like `json.dumps`, `datetime`, `decimal` — subtle float/formatting changes exist across minor versions
- Massive surface area

**Recommended starting point:** Prettier v2 vs v3 on the React or Next.js source tree. The formatting output differences are significant and immediately visible, which makes it a great demo and validation case for your tool.
